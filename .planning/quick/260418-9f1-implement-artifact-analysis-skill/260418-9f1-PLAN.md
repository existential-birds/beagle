<planning_decisions>
Spec §Open Questions resolved here so the executor does not second-guess.

**OQ-1. Slug derivation rule.** When `intent` is present, slug it per the same kebab rule as web-research (lowercased, punctuation stripped, whitespace → hyphens, truncated to 60 chars on word boundary). When `intent` is absent, derive from the first scanned path's basename. Always prefix with `YYYY-MM-DD-` for stable sort and idempotent re-run on the same day.

**OQ-2. Skip-pattern denylist.** Concrete glob list finalized in `references/skip-patterns.md`: `.env*`, `.git/`, `node_modules/`, `.venv/`, `venv/`, `__pycache__/`, `dist/`, `build/`, `target/`, `*.pem`, `*.key`, `*.p12`, `*.pfx`, and binary extensions (`*.png`, `*.jpg`, `*.jpeg`, `*.gif`, `*.webp`, `*.pdf`, `*.zip`, `*.tar`, `*.gz`, `*.mp4`, `*.mov`, `*.mp3`, `*.wav`, `*.sqlite`, `*.db`). First version reads plain text and markdown only — PDF/binary parsing is a future `beagle-core:docling` extension per spec §Out of scope.

**OQ-3. Subagent failure observability.** Same rule as web-research: every subagent writes at least a stub `findings/<slice-slug>.md` with `status:` frontmatter (`ok`, `empty`, `failed`) and a one-line `reason:` when not `ok`. Absence of the file after dispatch = silent failure, recorded under `Gaps & Limitations`.

**OQ-4. Re-run behavior when `output_dir` already contains a prior run.** Same rule as web-research: refuse by default; require explicit `refresh: true` to archive the prior run into `<output_dir>/.archive-<timestamp>/` and start fresh. Preserves the audit trail per spec Constraints.

**OQ-5. Trigger keyword calibration.** Frontmatter description triggers on local-document intent only — "analyze these docs", "scan my project", "read the docs folder", "summarize what's in .beagle/concepts/", "extract context from docs/". Explicit negative examples: codebase lookups (Grep/Glob), web research (that's web-research), credibility adjudication (that's llm-judge).

**OQ-6. `plan_gate` parameter.** Defer per spec §Open Questions "add only when needed" — do NOT add now. Skill writes `plan.md` for auditability but proceeds without a pause by default. A future caller can add the parameter with a worked example in the companion contract.
</planning_decisions>

3. Workflow — write `plan.md` → dispatch 1-3 parallel subagents → synthesize `report.md`. No plan gate. Small ASCII flow diagram.
4. Inputs — `intent` (string, optional), `paths` (list, optional), `output_dir` (absolute path, optional), `refresh` (bool, default false). Failure modes — partial-success on subagent failure; empty-corpus returns cleanly with "no documents found" under Gaps & Limitations (never errors); silent-failure detection via stub files; re-run protection. **subagent-brief.md** (~80-120 lines) — mechanical template filled per-slice. Fields: `Slice:`, `Intent:` (from caller; "generic salient extraction" when absent), `Paths to scan:` (bullet list), `What to extract:` (category cues from spec §Requirements), `Skim strategy:` (read index first for sharded docs; TOC/exec-summary/headings first for large docs), `Skip patterns:` (link to skip-patterns.md), `Output path:` (`<output_dir>/findings/<slice-slug>.md`), `Required frontmatter:` (`status:`, `slice:`, `brief_hash:`, `started_at:`, `finished_at:`, `paths_read:`, `paths_skimmed:`, `paths_skipped:`), `Citation rules:` (link to citation-schema.md), `Partial-failure protocol:` always write stub on failure, `Return:` one terse status line only.

**citation-schema.md** (~50-80 lines) — required: `path` (relative to scanned root when possible), `excerpt` (verbatim). Optional: `lines` (line range or single line — only when naturally available; never synthesized), `heading` (nearest enclosing heading), `document_type`. Footnote convention: `[^n]`; numbered `Sources` section at report bottom. Example well-formed citation block. Incomplete-metadata policy: omit rather than fabricate.

**report-template.md** (~80-120 lines) — literal skeleton for `report.md` with seven fixed sections in the spec-mandated order: `Documents Found`, `Key Insights`, `User / Market Context`, `Technical Context`, `Ideas & Decisions` (each tagged `accepted` / `rejected` / `open` with rationale), `Raw Detail Worth Preserving`, `Gaps & Limitations`. Rules: every section required every time; every claim carries a `[^n]` footnote with citation per schema; `Gaps & Limitations` required even when scan looks complete.

**failure-modes.md** (~80-120 lines) — four cases: partial-success (continue with remaining findings, enumerate failed slice under Gaps), empty-corpus (write `report.md` with "no documents found" under Gaps, no subagents spawned, return cleanly), silent-failure detection (stub file with `status:` frontmatter; absence = silent failure), re-run protection (refuse unless `refresh: true`; archive prior run into `.archive-<timestamp>/`). Explicit note that artifact-analysis does NOT fail-fast on missing tools — filesystem is assumed present. Verification checklist at end.

**companion-contract.md** (~100-140 lines) — input semantics table, success/refusal return shapes, worked examples:
- `prfaq-beagle` Ignition: `intent: "competitive and technical grounding for PRFAQ"`, `output_dir: .beagle/concepts/<slug>/analysis/`, `paths: []` (auto-discover).
- `brainstorm-beagle`: `intent: "reference points for idea-shaping"`, auto-discover.
- `strategy-interview`: `intent: "background context for strategy discovery"`, explicit `paths: ["docs/", ".beagle/concepts/"]`.
- Standalone user: `intent` omitted, `paths: ["docs/"]` — generic salient extraction.

Non-obligations mirror web-research: no intent reshaping, no coaching, no inline findings, no cross-run caching.

**skip-patterns.md** (~40-60 lines) — three groups with explicit glob lists: sensitive (`.env*`, `*.pem`, `*.key`, `*.p12`, `*.pfx`, `.aws/`, `.ssh/`), binary/media (images, archives, audio, video, db files), vendor/build (`node_modules/`, `.venv/`, `venv/`, `__pycache__/`, `dist/`, `build/`, `target/`, `.git/`, `coverage/`). Rule: subagents skip these silently and record skipped paths under `paths_skipped` frontmatter so the audit trail shows what was excluded.
